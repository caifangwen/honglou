## 方法二：通过游戏注册功能创建账号

如果方法一不行，可以使用游戏内的注册功能：

1. 在登录界面点击"注册"按钮
2. 输入邮箱和密码（密码至少6位）
3. 注册成功后会自动创建角色

注意：使用 `@example.com` 结尾的邮箱会自动创建测试角色。

## 方法三：使用 Supabase Auth API 创建账号

如果上述方法都不行，可以通过 Supabase Dashboard 手动创建用户：

1. 进入 Supabase Dashboard
2. 点击左侧菜单的 "Authentication" -> "Users"
3. 点击 "Add user" 按钮
4. 输入邮箱和密码
5. 勾选 "Auto Confirm User"（自动确认用户）
6. 创建用户后，需要在 SQL Editor 中运行以下 SQL 来创建对应的 player 记录：


## 调试联系人列表问题

我已经在 `ComposePanel.gd` 中添加了调试日志。如果联系人列表为空，请：

1. 打开 Godot 编辑器
2. 运行游戏并登录
3. 打开信箱，点击"写信"按钮
4. 查看 Godot 的输出面板（Output），会显示：
   - 当前的 game_id 和 player_db_id
   - 查询结果
   - 找到的联系人列表

### 常见问题：

**问题1：current_game_id 为空**
- 原因：登录后没有正确加载玩家数据
- 解决：检查 `Login.gd` 的 `_on_player_check()` 方法是否正确执行

**问题2：查询返回空数组**
- 原因：数据库中没有其他玩家，或者其他玩家的 current_game_id 不同
- 解决：确保运行了 seed.sql，并且所有测试玩家的 current_game_id 都是 `00000000-0000-0000-0000-000000000001`

**问题3：查询失败（code != 200）**
- 原因：数据库连接问题或权限问题
- 解决：检查 Supabase 配置和 RLS 策略

## 验证数据库数据

在 Supabase SQL Editor 中运行以下查询，检查测试数据是否正确：

```sql
-- 检查游戏局
SELECT * FROM public.games WHERE id = '00000000-0000-0000-0000-000000000001';

-- 检查玩家
SELECT id, auth_uid, username, display_name, character_name, role_class, current_game_id
FROM public.players
WHERE current_game_id = '00000000-0000-0000-0000-000000000001';

-- 检查认证用户
SELECT id, email, email_confirmed_at
FROM auth.users
WHERE email LIKE '%@example.com' OR email LIKE '%@test.com';
```

如果这些查询返回空结果，说明 seed.sql 没有正确执行。
