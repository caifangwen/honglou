# 听壁脚模块完善说明

## 概述

本次完善为《红楼回忆志》游戏的听壁脚（挂机监听）系统，新增了以下功能：

1. **EavesdropManager 增强** - 核心逻辑完善
2. **EavesdropScene 改进** - UI 交互优化
3. **IntelBag 筛选系统** - 情报管理增强
4. **EavesdropHub 挂机中心** - 会话管理界面
5. **IntelIntercept 情报拦截** - 管家专属功能
6. **数据库迁移** - 表结构优化

---

## 一、EavesdropManager.gd 完善

### 新增功能

#### 1. 信号系统
```gdscript
signal session_started(session_id: String)
signal session_completed(session_id: String, intel_count: int)
signal intel_received(content: String, value_level: int)
```

#### 2. 双人挂机奖励
```gdscript

```

#### 3. 自动定时器管理
- 会话完成检查定时器（每 30 秒）
- 活跃会话恢复机制
- 自动清理已完成会话的定时器

#### 4. 场景配置增强
每个场景现在包含 `description` 字段，用于 UI 展示。

### 核心方法

| 方法 | 说明 |
|------|------|
| `start_eavesdrop()` | 开始挂机，检查精力和人数 |
| `generate_intel_fragment()` | 生成情报，支持双人奖励 |
| `check_session_completion()` | 检查并完成会话 |
| `get_active_sessions()` | 获取活跃会话列表 |
| `cancel_session()` | 取消会话 |
| `get_session_time_remaining()` | 获取剩余时间 |

### 成功率计算

```
最终触发率 = 基础触发率 × 人数修正 + 双人奖励

人数修正:
- 1-2 人：100%
- 3-4 人：80%
- 5 人以上：50%
```

## 五、IntelIntercept 情报拦截

### 文件结构

```
scenes/ui/IntelIntercept.tscn
scripts/ui/IntelIntercept.gd
```

### 功能特性

#### 1. 管家专属能力
只有管家角色的玩家可以使用情报拦截功能。

#### 2. 拦截机制
- 消耗 5 点精力
- 选择目标玩家
- 设置拦截时长（1-12 小时）
- 被拦截玩家无法获得新的情报碎片

#### 3. 拦截检查
```gdscript
# EavesdropManager 在生成情报前检查
var intercept_res = await SupabaseManager.db_get(
    "/rest/v1/intel_intercepts?target_uid=eq.%s&status=eq.active&ends_at=gt.now&select=id" % player_uid
)
if intercept_res["code"] == 200 and not intercept_res["data"].is_empty():
    return  # 被拦截，不生成情报
```

### 使用流程

1. 从 Hub 进入「情报拦截」
2. 选择目标玩家
3. 设置拦截时长
4. 确认执行，扣除精力

## 八、使用流程

### 开始挂机

1. 从 Hub 场景进入「挂机监听」
2. 选择场景（查看描述和预期收益）
3. 选择时长（1-8 小时）
4. 如有搭档，可选择双人挂机
5. 确认开始，扣除 2 点精力

### 查看情报

1. 进入「情报背包」
2. 使用筛选功能查找情报
3. 点击查看详情
4. 可选择：
   - 出售给其他玩家
   - 发布流言
   - 管家可封锁情报

### 管理会话

1. 进入「挂机中心」
2. 查看所有活跃会话
3. 实时查看剩余时间和收益
4. 可取消不需要的会话

## 十二、总结

本次完善为听壁脚模块添加了完整的游戏循环：

```
开始挂机 → 等待完成 → 获得情报 → 筛选管理 → 使用/出售
     ↑                                          ↓
     └────────────── 挂机中心管理 ──────────────┘
     
管家专属：情报拦截功能
```

所有代码遵循项目现有规范，数据库迁移脚本可直接应用于生产环境。
