# 挂机功能调试说明

## 问题诊断

挂机功能提示"网络问题或精力不足"可能有以下原因：

### 1. 精力不足
- 挂机需要消耗 **2 点精力**
- 精力每 **2 小时（7200 秒）** 自动恢复 1 点
- 管家精力上限：6 点
- 丫鬟精力上限：8 点

### 2. 网络/数据库问题
- 本地数据库未启动
- pgREST 服务未运行
- 数据库表结构不正确

## 调试工具

### 调试精力按钮

在挂机场景的顶部工具栏添加了 **"调试：精力 +3"** 按钮。

**使用方法：**
1. 打开挂机场景
2. 点击"调试：精力 +3"按钮
3. 查看弹出的调试信息对话框
4. 点击"确定"补充 3 点精力

**调试信息显示：**
- 当前精力值 / 最大精力值
- 挂机消耗精力
- 最后刷新时间（秒/分钟）
- 预计恢复时间（小时）


精力不足时标签会变红，按钮显示"精力不足"。

## 快速解决方案

### 方案 1：使用调试按钮（推荐）

1. 打开挂机场景
2. 点击"调试：精力 +3"
3. 确认补充精力
4. 开始挂机

### 方案 2：等待自动恢复

- 每 2 小时自动恢复 1 点精力
- 查看控制台中的"自动恢复：X 小时后"提示

### 方案 3：检查数据库

如果提示网络问题：

```powershell
# 检查 Docker 容器状态
docker ps | findstr "honglou"

# 重启 pgREST 服务
docker restart honglou_pgrest

# 查看数据库日志
docker logs honglou_local_db --tail 20
```

## 常见问题

### Q1: 精力显示为 0 怎么办？

**A:** 点击"调试：精力 +3"按钮补充精力，或等待自动恢复。

### Q2: 点击开始挂机没有反应？

**A:** 查看控制台输出，检查是否有以下错误：
- `精力不足` - 使用调试按钮补充精力
- `current_game_id is empty` - 游戏状态未初始化，返回主界面重新进入
- `插入会话到数据库失败` - 检查数据库连接

### Q3: 调试按钮无效？

**A:** 检查控制台是否有错误信息。如果 `debug_dialog` 节点不存在，会直接增加精力。

### Q4: 精力扣除失败？

**A:** 可能是 `PlayerState.consume_stamina()` 返回 false。检查：
- 精力是否足够
- `last_stamina_refresh` 是否正确设置
- 控制台是否有警告信息
