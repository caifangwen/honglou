## 快速开始

### 1. 启动本地数据库（开发模式）

```powershell
# 使用 pgREST HTTP API（推荐）
docker-compose -f docker-compose.local-dev.yml up -d

# 或仅数据库（无 HTTP API）
docker-compose up -d
```

### 2. 在 Godot 中切换模式

修改 `.env.local` 文件：

```env
# 本地开发模式
USE_LOCAL_DB=true

# 云端 Supabase 模式
USE_LOCAL_DB=false
```

### 3. 启动 Godot 项目

- **方式一**：在 Godot 编辑器中直接运行（默认读取 `.env.local`）
- **方式二**：使用命令行参数启动
  ```powershell
  # 本地模式
  godot --use-local-db=true
  
  # 云端模式
  godot --use-local-db=false
  ```

## 切换模式

### 方法一：使用 .env.local 文件（推荐）

1. 在项目根目录创建 `.env.local` 文件：
   ```env
   USE_LOCAL_DB=true
   ```

2. 在 Godot 中运行项目，会自动读取配置

### 方法二：命令行参数

```powershell
# 本地开发
godot --use-local-db=true

# 云端部署
godot --use-local-db=false
```

### 方法三：修改 GameConfig.gd（永久切换）

```gdscript
# 在 _init_environment() 中硬编码
func _init_environment() -> void:
    USE_LOCAL_DB = true  # 强制本地模式
    # USE_LOCAL_DB = false  # 强制云端模式
```

## 架构说明

### 本地开发模式

```
┌─────────────┐     HTTP      ┌─────────────┐
│   Godot     │ ────────────> │   pgREST    │
│   游戏客户端 │    3000 端口   │  REST API   │
└─────────────┘               └──────┬──────┘
                                     │
                                     ▼
                               ┌─────────────┐
                               │  PostgreSQL │
                               │   5432 端口  │
                               └─────────────┘
```

**组件说明**：
- **PostgreSQL**: 数据库，端口 5432
- **pgREST**: HTTP API 中间件，端口 3000
- **pgAdmin**: 数据库管理界面，端口 5050

**访问地址**：
- pgAdmin: http://localhost:5050 (admin@admin.com / admin)
- pgREST API: http://localhost:3000
- PostgreSQL: localhost:5432

### 云端 Supabase 模式

```
┌─────────────┐     HTTPS     ┌─────────────┐
│   Godot     │ ────────────> │  Supabase   │
│   游戏客户端 │               │    云端     │
└─────────────┘               └─────────────┘
```

## 模式对比

| 功能 | 本地模式 | 云端 Supabase |
|------|----------|--------------|
| **数据库** | PostgreSQL 15 | PostgreSQL 15 |
| **用户认证** | ❌ 模拟 | ✅ 完整 Auth 系统 |
| **RLS 安全** | ❌ 跳过 | ✅ 启用 |
| **Realtime** | ❌ 不支持 | ✅ WebSocket |
| **Storage** | ❌ 不支持 | ✅ 文件存储 |
| **适用场景** | 单机开发、逻辑测试 | 多人联机、完整测试 |

## 推送到 Supabase

当你准备将本地开发的功能推送到云端：

### 1. 登录 Supabase CLI

```powershell
supabase login
```

### 2. 关联项目

```powershell
cd "C:\Users\Frida\红楼回忆志"
supabase link --project-ref daotqqwsxvydxqttmams
```

### 3. 推送数据库结构

```powershell
supabase db push
```

### 4. 切换回云端模式

修改 `.env.local`：
```env
USE_LOCAL_DB=false
```

## 常见问题

### Q1: 本地模式无法登录？

**原因**：数据库中没有对应的玩家记录

**解决**：
1. 检查数据库是否启动：`docker-compose ps`
2. 检查测试数据是否加载：
   ```sql
   SELECT * FROM public.players;
   ```
3. 重新初始化数据库：
   ```powershell
   docker-compose -f docker-compose.local-dev.yml down
   rmdir /s /q local_db_data
   docker-compose -f docker-compose.local-dev.yml up -d
   ```

### Q2: 如何查看本地数据库数据？

**方式一**：使用 pgAdmin
1. 浏览器访问 http://localhost:5050
2. 登录：admin@admin.com / admin
3. 连接数据库：honglou

**方式二**：使用命令行
```powershell
docker exec -it honglou_local_db psql -U postgres -d honglou
```

### Q3: 本地模式能测试 Realtime 功能吗？

不能。Realtime 是 Supabase 特有的功能，依赖 WebSocket 和 PostgreSQL 触发器。

**建议**：
- 本地测试基础游戏逻辑
- Realtime 功能留到云端测试

### Q4: 如何在本地测试用户注册？

本地模式没有 Auth 系统，有两种方案：

1. **模拟注册**：代码自动创建玩家记录
2. **连接云端 Auth**：本地游戏连接云端 Supabase Auth

```env
LOCAL_DB_PORT=5432
LOCAL_DB_NAME=honglou
LOCAL_DB_USER=postgres
LOCAL_DB_PASSWORD=postgres123
```

### docker-compose.local-dev.yml

包含三个服务：
- `postgres`: PostgreSQL 数据库
- `pgrest`: REST API 中间件
- `pgadmin`: 数据库管理界面

```

3. 推送云端 → supabase db push
```
