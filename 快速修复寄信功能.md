# 快速修复寄信功能

## 问题诊断

对比 8963c34 提交后发现，当前代码无法发送信息的原因是：

**`messages` 表缺少必要的字段**

8963c34 提交时的代码可以正常工作，但后续的数据库 schema 更新没有同步 `messages` 表的结构变更。

## 一键修复命令

### Windows PowerShell（推荐）

```powershell
docker exec -i honglou_local_db psql -U postgres -d honglou -f fix_messages_schema.sql
```

或者手动执行：

```powershell
Get-Content fix_messages_schema.sql | docker exec -i honglou_local_db psql -U postgres -d honglou
```

### Windows CMD

```cmd
type fix_messages_schema.sql | docker exec -i honglou_local_db psql -U postgres -d honglou
```

## 根本原因分析

### 代码变更

1. **GameConfig.gd**: 添加了本地/云端双模式支持
2. **SupabaseManager.gd**: 添加了本地模式适配逻辑
3. **InboxManager.gd**: 添加了调试日志

### 数据库变更

`messages` 表在 8963c34 提交后没有同步更新，缺少以下字段：
- `message_type`
- `attachments`
- `stamina_cost`
- `is_tampered`
- `original_content`
- `is_intercepted`
- `stage`
- `expires_at`

### 修复方案

1. 更新 `schema.sql` 和 `combined_schema.sql` 中的 `messages` 表定义
2. 创建 `fix_messages_schema.sql` 迁移脚本
3. 更新 `docker-compose.local-dev.yml` 自动应用迁移
4. 在 `SupabaseManager.gd` 中添加调试日志
