# 挂机问题诊断指南

## 第一步：运行游戏并查看控制台

启动游戏后，打开挂机场景，**不要点击任何按钮**，观察控制台输出。

## 第二步：点击"调试：精力 +3"按钮

点击后应该看到：
1. 弹出调试信息对话框
2. 显示当前精力状态
3. 点击"确定"后精力增加 3 点

**如果无效**：检查控制台是否有错误信息

## 第三步：选择一个场景

点击任意场景卡片（如"怡红院后窗"），然后观察控制台：

```
[EavesdropScene] Scene yi_hong_yuan count: 0
```

**正常情况**：显示每个场景的人数
**异常情况**：显示错误信息

## 第四步：点击"开始挂机"

### 常见错误及解决方案

#### 错误 1：PlayerState.uid 为空

```
[EavesdropScene] PlayerState.uid: 
```

**原因**：登录流程未完成或 PlayerState 未正确初始化

**解决**：
1. 返回登录界面重新登录
2. 使用测试账号快速登录
3. 检查 Login.gd 中的 _on_auth_success 是否被调用

#### 错误 2：精力不足

```
[EavesdropScene] 精力检查：当前=0, 需要=2
[EavesdropScene] 精力不足！stamina=0, get_current_stamina()=0
```

**解决**：点击"调试：精力 +3"按钮补充精力

#### 错误 3：数据库返回错误

```
[Eavesdrop] 数据库返回：code=400, data={...}
```

**可能原因**：
- 表结构不正确
- 外键约束失败（player_uid 不存在）
- RLS 权限问题

**解决**：
```powershell
# 检查 eavesdrop_sessions 表
docker exec -i honglou_local_db psql -U postgres -d honglou -c "\d eavesdrop_sessions"

# 检查 players 表
docker exec -i honglou_local_db psql -U postgres -d honglou -c "SELECT id, character_name FROM players;"

# 重启 pgREST
docker restart honglou_pgrest
```

#### 错误 4：场景人数查询失败

```
[EavesdropManager] get_scene_listener_count failed: code=404
```

**原因**：game_id 无效或表不存在

**解决**：
```powershell
# 检查 games 表
docker exec -i honglou_local_db psql -U postgres -d honglou -c "SELECT id, status FROM games;"

# 如果没有数据，插入测试数据
docker exec -i honglou_local_db psql -U postgres -d honglou -c "INSERT INTO games (id, status) VALUES ('00000000-0000-0000-0000-000000000001', 'active') ON CONFLICT (id) DO NOTHING;"
```

## 第五步：查看挂机中心

如果挂机成功，返回挂机中心应该看到：

```
[EavesdropHub] _load_active_sessions called
[EavesdropHub] get_active_sessions returned: [{...}]
[EavesdropHub] Creating 1 session items
```

并且显示你的挂机会话卡片。

## 快速诊断命令

### 检查数据库连接

```powershell
# 测试数据库连接
docker exec -i honglou_local_db psql -U postgres -d honglou -c "SELECT count(*) FROM players;"

# 测试 pgREST
curl http://localhost:3000/players?select=id,character_name
```

### 检查 Docker 容器

```powershell
# 查看所有容器状态
docker ps | findstr "honglou"

# 查看 pgREST 日志
docker logs honglou_pgrest --tail 20

# 查看数据库日志
docker logs honglou_local_db --tail 20
```

### 手动插入测试会话

如果无法通过 UI 创建会话，可以手动测试：

```sql
-- 手动插入测试会话
INSERT INTO eavesdrop_sessions (
    player_uid, game_id, scene, scene_key, status, ends_at
) VALUES (
    '11111111-1111-1111-1111-111111111111',
    '00000000-0000-0000-0000-000000000001',
    'yi_hong_yuan',
    'yi_hong_yuan',
    'active',
    now() + interval '1 hour'
);

-- 检查是否插入成功
SELECT id, player_uid, scene, status FROM eavesdrop_sessions LIMIT 5;
```

## 报告问题

如果问题仍然存在，请提供以下信息：

1. **控制台完整日志**（从点击开始挂机到错误出现）
2. **PlayerState 状态**（从"状态检查"输出中复制）
3. **数据库返回的错误代码**（code=xxx）
4. **错误对话框内容**（如果有）

这样可以更快地定位问题！
