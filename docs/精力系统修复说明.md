# 精力系统修复说明

> 修复日期：2026-03-18  
> 问题：精力数值加减不符合预期

---

## 问题分析

### 原问题

1. **`stamina` setter 每次都会重置 `last_stamina_refresh`**
   - 导致每次设置精力值时，精力恢复计时器都会重置
   - 造成精力恢复时间计算错误

2. **多处代码直接使用 `stamina -=` 而不是 `consume_stamina()`**
   - 绕过了正确的精力计算逻辑
   - 没有更新 `last_stamina_refresh`
   - 没有同步到数据库

3. **精力恢复计算可能重复**
   - `get_current_stamina()` 会计算恢复的精力
   - 但如果 `last_stamina_refresh` 设置不正确，会导致恢复计算错误

---

## 修复内容

### 1. 修改 `PlayerState.gd` 中的 `stamina` setter

**修复前**：
```gdscript
var stamina: int = 6:
    set(val):
        if stamina != val:
            stamina = val
            # ❌ 每次设置都重置刷新时间
            last_stamina_refresh = int(Time.get_unix_time_from_system())
            stamina_changed.emit(stamina)
```

**修复后**：
```gdscript
var stamina: int = 6:
    set(val):
        if stamina != val:
            stamina = val
            # ✅ 只发出信号，不重置刷新时间
            stamina_changed.emit(stamina)
```

### 2. 修复 `consume_stamina()` 方法

**修复前**：
```gdscript
func consume_stamina(amount: int) -> bool:
    var current: int = get_current_stamina()
    if current < amount:
        return false
    stamina = current - amount
    sync_to_db()
    return true
```

**修复后**：
```gdscript
func consume_stamina(amount: int) -> bool:
    var current: int = get_current_stamina()
    if current < amount:
        print("[PlayerState] 精力不足：当前=%d, 需要=%d" % [current, amount])
        return false
    # ✅ 直接设置基础精力值
    stamina = current - amount
    # ✅ 更新刷新时间，避免重复恢复
    last_stamina_refresh = int(Time.get_unix_time_from_system())
    print("[PlayerState] 精力扣除成功：扣除=%d, 剩余基础精力=%d" % [amount, stamina])
    sync_to_db()
    return true
```

#### DebugPanelUI.gd

# 修复前
if PlayerState.stamina >= act.cost:

## 精力计算逻辑

### 正确流程

1. **获取当前精力**（包含自动恢复）：
   ```gdscript
   var current := PlayerState.get_current_stamina()
   # 计算公式：min(基础精力 + 恢复次数，精力上限)
   # 恢复次数 = (当前时间 - 最后刷新时间) / 7200 秒
   ```

2. **消耗精力**：
   ```gdscript
   PlayerState.consume_stamina(5)
   # 1. 计算当前精力（含恢复）
   # 2. 检查是否足够
   # 3. 更新基础精力 = 当前精力 - 消耗
   # 4. 重置刷新时间为当前时间
   # 5. 发出信号并同步到数据库
   ```

3. **手动增加精力**（如调试、奖励）：
   ```gdscript
   var current := PlayerState.get_current_stamina()
   PlayerState.stamina = min(current + 3, PlayerState.stamina_max)
   PlayerState.last_stamina_refresh = int(Time.get_unix_time_from_system())
   PlayerState.stamina_changed.emit(PlayerState.stamina)
   ```

## 验证方法

### 1. 单元测试

```gdscript
# 测试精力消耗
func test_stamina_consume():
    PlayerState.stamina = 6
    PlayerState.last_stamina_refresh = int(Time.get_unix_time_from_system())
    
    var success = PlayerState.consume_stamina(2)
    assert(success == true)
    assert(PlayerState.stamina == 4)  # 基础精力应为 4
    
# 测试精力不足
func test_stamina_insufficient():
    PlayerState.stamina = 1
    PlayerState.last_stamina_refresh = int(Time.get_unix_time_from_system())
    
    var success = PlayerState.consume_stamina(5)
    assert(success == false)  # 应该失败
    
# 测试精力恢复
func test_stamina_recovery():
    PlayerState.stamina = 3
    PlayerState.last_stamina_refresh = int(Time.get_unix_time_from_system()) - 7200 * 2  # 2 小时前
    
    var current = PlayerState.get_current_stamina()
    assert(current == 6)  # 应恢复到上限 6
```

## 注意事项

1. **不要直接使用 `stamina -=` 或 `stamina +=`**
   - 始终使用 `consume_stamina()` 消耗精力
   - 如需增加精力，手动更新后记得设置 `last_stamina_refresh`

2. **精力恢复是自动计算的**
   - 不需要定时器或手动触发
   - `get_current_stamina()` 会自动计算恢复量

3. **数据库同步**
   - `consume_stamina()` 会自动同步到数据库
   - 手动修改精力后需要调用 `sync_to_db()`
