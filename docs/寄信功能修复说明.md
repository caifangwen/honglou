# 寄信功能修复说明

## 问题原因

8963c34 提交时，`messages` 表的结构不完整，缺少以下字段：
- `message_type` - 消息类型（私信/流言/批条等）
- `attachments` - 附件（如情报碎片）
- `stamina_cost` - 精力消耗
- `is_tampered` - 是否被丫鬟篡改
- `original_content` - 原始内容
- `is_intercepted` - 是否被丫鬟截留
- `stage` - 流言发酵阶段
- `expires_at` - 过期时间

这导致 `InboxManager.send_private_message()` 尝试插入数据时失败。

## 修复步骤

### 方案一：手动执行 SQL 修复（推荐）

1. 执行修复脚本：
```bash
docker exec -i honglou_local_db psql -U postgres -d honglou < fix_messages_schema.sql
```

2. 验证表结构：
```bash
docker exec -i honglou_local_db psql -U postgres -d honglou -c "\d messages"
```

### 方案二：重启 Docker 容器（自动应用最新 schema）

1. 停止并删除现有容器：
```bash
docker-compose -f docker-compose.local-dev.yml down
```

2. 删除本地数据库数据（可选，会清空所有数据）：
```bash
rm -rf local_db_data
```

3. 重新启动容器（会自动使用更新后的 combined_schema.sql）：
```bash
docker-compose -f docker-compose.local-dev.yml up -d
```

## 代码修复

### SupabaseManager.gd

修复了 `_build_headers` 方法，在本地模式下添加正确的 apikey 头：

```gdscript
func _build_headers(with_auth):
    var headers = PackedStringArray([
        "Content-Type: application/json",
        "Prefer: return=representation"
    ])

    if _is_local_mode:
        headers.append("apikey: local-dev-key")
        headers.append("Prefer: return=representation,merge-duplicates")
    else:
        headers.append("apikey: " + GameConfig.SUPABASE_ANON_KEY)
        if with_auth and access_token != "":
            headers.append("Authorization: Bearer " + access_token)

    return headers
```

## 常见问题

### 问题 1：仍然收到 400/401 错误

**原因**：pgREST 服务的 anon 角色权限不足

**解决方法**：
```bash
docker exec -i honglou_local_db psql -U postgres -d honglou < local_dev_permissions.sql
```

### 问题 2：容器启动失败

**原因**：端口冲突或数据目录权限问题

**解决方法**：
```bash
# 检查端口占用
netstat -ano | findstr "3000\|5432\|5050"

# 停止冲突的进程或修改 docker-compose.yml 中的端口映射
```

### 问题 3：数据库表不存在

**原因**：初始化 SQL 未正确执行

**解决方法**：
```bash
# 查看容器日志
docker logs honglou_local_db

# 手动执行 schema
docker exec -i honglou_local_db psql -U postgres -d honglou < combined_schema.sql
```
