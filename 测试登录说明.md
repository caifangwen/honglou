# Godot 本地登录测试指南

## 前提条件

确保以下服务正在运行：

```powershell
# 检查容器状态
docker-compose -p honglou ps

# 如果未运行，启动服务
docker-compose -p honglou -f docker-compose.local-dev.yml up -d
```

1. 打开 Godot 编辑器

## 常见问题排查

### 问题 1：无法连接数据库

**错误**：`Connection refused` 或 `Network error`

**解决**：
```powershell
# 检查 pgREST 是否运行
docker-compose -p honglou logs pgrest

# 重启 pgREST
docker-compose -p honglou -f docker-compose.local-dev.yml restart pgrest

# 测试 API
curl http://localhost:3000/players?limit=1
```

### 问题 2：玩家数据为空

**错误**：`No player found`

**解决**：
```powershell
# 检查数据库中是否有测试数据
docker exec -it honglou_local_db psql -U postgres -d honglou -c "SELECT * FROM public.players;"

# 如果为空，重新初始化
docker-compose -p honglou -f docker-compose.local-dev.yml down
rmdir /s /q local_db_data
docker-compose -p honglou -f docker-compose.local-dev.yml up -d
```

### 问题 3：切换到云端模式测试

修改 `.env.local`：
```env
USE_LOCAL_DB=false
```

重启 Godot，此时会使用 Supabase 云端数据库。

## API 测试

使用 curl 测试本地 API：

```powershell
# 查询所有玩家
curl http://localhost:3000/players?select=id,username,display_name

# 查询凤姐
curl "http://localhost:3000/players?id=eq.11111111-1111-1111-1111-111111111111"

# 查询游戏局
curl http://localhost:3000/games?select=id,status,current_day
```
